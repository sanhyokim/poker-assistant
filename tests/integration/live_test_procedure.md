# Level 2 ライブ映像テスト手順書

## 前提条件

- ハードウェア: UGREEN キャプチャカード接続済み、RTX 3080搭載PC
- ソフトウェア: Python 3.11.9、全依存パッケージインストール済み
- CoinPoker: デスクトップクライアントインストール済み、Practice Gamesテーブルに着席可能
- 設定: `.env` に `OPENROUTER_API_KEY` が設定済み

## テスト環境セットアップ

1. CoinPokerクライアントを起動し、ウィンドウを最大化する
2. 4色デッキ設定が有効であることを確認（スペード黒、ハート赤、ダイヤ青、クラブ緑）
3. UI言語が日本語であることを確認
4. Practice Games の NLH 6max テーブル（5/10ブラインド推奨）に着席
5. `config.yaml` を以下の通り確認・設定:
   - `capture.method: capture_card`
   - `capture.device_index: （キャプチャカードのデバイス番号）`
   - `game.blind_sb: 5`（Practice Gamesの場合）
   - `game.blind_bb: 10`（Practice Gamesの場合）
   - `solver.rake_rate: 0.0`
   - `solver.rake_cap: 0.0`

## テスト実行手順

1. `python main.py` で起動
2. 稼働タブの START ボタンを押す
3. 30分間プレイする（自然なペースで。意図的に速く/遅くしない）
4. 以下を観察・記録する

## 判定基準

- **PASS**: 認識エラー0件/30分、ハンド境界10ハンド連続正確、フォールバック発動率 < 5%
- **CONDITIONAL PASS**: 認識エラー1-2件で原因特定済み、config調整で解決見込み
- **FAIL**: 認識エラー3件以上、またはハンド境界が不安定

## トラブルシューティング

- キャプチャカードが認識されない → `device_index` を0-9で試行
- OCR精度が低い → `logs/poker_assistant.log` のDEBUGレベルで詳細確認
- ソルバーが起動しない → `solver/bin/postflop_cli.exe` の存在確認、手動実行テスト
- LLMがタイムアウト → `.env` の `OPENROUTER_API_KEY` 確認、OpenRouter接続確認
